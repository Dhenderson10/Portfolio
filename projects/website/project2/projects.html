Semi-Autonomous RC car

Github Code

Details

This project was built from the ground up using various components. The first crucial piece to this project was an XMOS board, which was used as the main interface for the device. Attached to that were two servos motors, these motors controlled the two big rear wheels you can see in the pictures above. Attached to the XMOS board was a Wifi chip which was used to communicate to the device directly using telnet, where commands could be issued. Finally, the last piece intergrated was an IMU, which helps the car determine where it is by calculating the current angle of the device relative to its position when it was turned on. The car was created to complete a small maze with obstacles, we achieved a semi-autonomous solution to this challenge by using a breadth first search algorithm, and commands specific to the maze environment.

Edge Detection

Github Code

Details

Edge detection works by creating a seperate image that stores modified values for the RGB values of a pixel. It begins by taking the 8 pixels around one pixel (up, down, left, right, and diagnols) and calculating their greyscale value, which is an average of the RGB. Those greyscale values are then used to determine Vx and Vy, which is computed as a convolution/kernel. Once Vx and Vy are computed, they are used in the final formula for the center pixel you were modifying, which is grey = √ (Vx2+Vy2) . After that, one must set pixel = (255 - grey), which colors the pixels that are not edges white. Special cases are needed for the boundries, and were ignored in this project.
